Network Group Settings

This topic details all the settings for plans. This is the information you will need when you add or edit network groups.

Setting Description
Name Required. Names cannot be longer than 32 characters and can contain only letters, number, dashes, and underscores.
GUID A GUID (globally unique identifier) is generated automatically for a network group when it is first created and saved. The GUID cannot be changed.
SID/NID/BSID

This field contains identifiers for 3GPP2 base stations. The identifiers are entered as combinations of SIDs, NIDs, and BSIDs.

Valid Values

The following values of SIDs, Nids, and BSIDs are valid:

  • SID: 1 to 32767, inclusive
  • NID: 1 to 65535, inclusive
  • BSID: 1 to 65535, inclusive
Valid Combinations

When entered in combinations, each value must be separated by the / character. You can enter the * character as a wildcard for any value to represent the entire range. BSIDs can be entered as 2 values separated by the - character to define a specific range. The following combinations of SIDs, NIDs, and BSIDs are valid:

  • SID
  • SID/NID
  • SID/NID/BSID
  • SID/NID/BSID-BSID
  • */NID
  • SID/*/BSID
  • SID/*/BSID-BSID
  • SID/NID/*
  • */*/BSID
  • */*/BSID-BSID
MCC/MNC

This field contains identifiers for 3GPP base stations. The identifiers are entered as MCC/MNC (HNI) tuples.

Valid Values
  • MCC: Three-digit integers, 000 to 999, inclusive
  • MNC: Two- or three-digit integers, 00 to 999, inclusive
Valid Combinations

MCC and MNC values must be entered in combination, separated by the / character. You can enter the * character as a wildcard for the MNC to represent the entire range. The following combinations of MCCs and MNCs are valid:

  • MCC/MNC
  • MCC/*

A list of MCC and MNC codes can be found here: http://en.wikipedia.org/wiki/Mobile_country_code

Validating Data

When you enter data into the network group fields, {{tag.Product Name}} performs immediate validation of the data. This allows you to see that there has been invalid data entered and to fix the data immediately. Invalid data can include incorrect characters (such as letter), incorrect combinations, numbers out of valid ranges, and duplicate entries.

As you enter data and it gets validated, a status message at the top of the dialog box provides a running total of how many entries were made successfully (were valid) and how many had issues (were invalid).

Fixing Data Errors

When invalid data gets entered, {{tag.Product Name}} displays two types of feedback:

  • At the top of the dialog box, a message in red shows how many data issues were found.
  • The data itself is highlighted in the field with white text on a red background. 

You can:

  • Fix the errors: Click on the entry and enter corrected, valid data.
  • Clear errors: Click the Clear errors link. All invalid data entries are removed. 

Click Save when you finish fixing errors.

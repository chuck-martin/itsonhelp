Filters

Filters are used in components of service policies and determine whether the associated policy events in the component are enacted. The associated policy event(s) define the action taken when the filter is matched. 

The type of filter and the settings you can configure depend on the type of service policy: Data service policies have data filters, voice service policies have voice filters, and message service policies have message filters.

Data service policy filters include:

  • Domain
  • IP Address
  • Protocol
  • Port
  • App ID

Voice service policy filters include:

  • Remote Phone Number
  • Direction

Message service policy filters include:

  • Remote Phone Number
  • Direction
  • SMS Body

Filter Sample Scenarios

You have a data service policy that blocks all websites in a certain network group. However, you still want to allow your users to access your website. So you create a component in the policy where the policy event defines the Allow action in the network group, and in the data filter, you add your domain. 

You have a feature that blocks all device connectivity in a certain country, but you want to allow phone calls and text messages to your customer service and technical support. In the voice and text policies of the feature, you create components where the policy event defines the Allow event in the network group and filters where you create a regular expression that evaluates to the specific phone numbers.

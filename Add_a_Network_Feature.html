Add a Network Feature

  1. Go to Policy > Features.
  2. Add the feature:
    1. Click Create New Feature, then Publishable Feature.
    2. Enter a Feature Name. Names cannot be longer than 32 characters and can contain only letters, number, dashes, and underscores.
    3. Enter a unique Feature Code. Consider carefully when creating a feature code. It is used throughout the subscriber provisioning system to identify the network feature and cannot be changed once you create the feature.
    4. Decide whether you want embedded-client device users to have the ability to enable and disable the network feature. If you select the Device users can enable/disable this feature check box, select the Default User State:
      • On: The network feature is enabled when it is pushed to the embedded-client device.
      • Off (default): The network feature is disabled when it is pushed to the embedded-client device. 
    5. If you allow embedded-client device users to enable/disable the network feature, select a UX Reference Code. (See Network Feature Properties for details about this property.)
    6. Optionally, enter a Description. A good description will explain what the feature is intended to do.
    7. Click Save.
  3. Add one or more policies to the feature:
    • Add a new or cloned service policy.
    • Add a new or cloned entitlement policy.
    • Add a data session policy.
    • Add a WiFi policy.
  4. Optionally select a feature Priority.
Changes are saved automatically as you make them. However, device behavior is not enforced by network features until you test and publish the network feature.

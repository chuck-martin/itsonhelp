Add a Policy Event

You add policy events either directly in policies or within components of policies.

  1. Click Add Policy Event > New.
  2. Select the Action:
    • Allow
    • Block
  3. Add network groups

    • In the IN or NOT IN (any of these Network Group(s)) field, click Add  , then click From Library to use an already defined network group (the recommended option) or New to create a new network group.

      In general, you should add network groups from your Network Group library. Too many ad hoc network groups can be difficult to manage. If the collection of network access points you want to use in the policy event does not exist in a defined network group, consider adding a new network group to the network group library with the collection of network access points you want to use, and then adding that new network group to the policy event from the network group library.
      • From Library: In the Subscriber Group list, select a network group, and then click Add.

      • New: See Add a Network Group for details on creating new network groups.

  4. Add one or more criteria. For each policy event, you can add one each of up to four criteria:
    1. Click Add Criteria.
    2. Select:
      • Network Type (data service policies only): Select from 2G, 3G, 4G, Cellular Network (default, any 2G, 3G, or 4G), or WiFi
      • App Visibility (data service policies only): Select from Background (default, an app running but not visible on the device) or Foreground or Streaming (a visible running app or an app that's streaming, whether it's in the foreground or background).
      • Time of Day: Select Start and Stop times and days or select Weekdays or Weekends.
        Devices synchronize their clocks by getting the time zone they are in from the nearest cell tower, and then using the current time information they get from the ItsOn NTP server for policy enforcement, so attempts to adjust the device clock to get service outside of any times defined by policy will not be successful.
      All the Criteria must be true for the Action to be applied.

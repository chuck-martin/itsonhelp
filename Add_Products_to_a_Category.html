Add Products to a Category

There is an extra consideration if you're adding a product to an exchange category. When you add a product to an exchange category, the product you add must be compatible with any products that are already in the exchange category, or else you will not be able to promote the change that contains the added product. See Changes to Products in Exchange Categories for more information.
  1. Select the Sandbox view you want to work in. Hover over the down arrow next to the current sandbox name in the header, then click a sandbox name.
  2. Go to Catalog > Categories.
  3. Click a category name.
  4. Click the Products tab.
  5. Click Add.
  6. Click a product name.
  7. Enter a number for Display Order. The number in the Display Order field determines the order in which products are displayed in a category, starting with the product that has lowest number, and then products in ascending order of the numbers in the Display Order field.
    If more than one product in a category has the same number in the Display Order field, the ordering of those products is determined by an internal product identifier, with the product with the smallest identifier being displayed first.
  8. Click Save.

The product gets added to the category. To add additional products, repeat steps 5 through 8.

Fees, eMoney, external products, preferred destination change fees, and basic bundles are not allowed in exchange categories. If any of these products are in an exchange category when you promote a change that includes that exchange category, the promotion will not be allowed and you will see an error.

Service Design Center uses a change management process to track and approve all changes to any product. All of these changes, including additions, edits, and deletions, must go through the promote-approve-deploy process before they become available.

Any changes you make (change a name, change a price, add or delete a product from a bundle, etc.) to any product does not affect any users that already have the product.

When you promote a change to a carousel exchange category, Service Design Center validates the products in the exchange category to make sure all the products are compatible for exchange. If there is an incompatibility, the promotion will not be allowed. The error message associated with the failed promotion will help in troubleshooting the issue. See Changes to Products in Exchange Categories for more information.

Publish a Network Feature or eCommerce Policy

Publishing a network feature means adding the capability of the ItsOn network to enforce the network feature's policies and equipping your customers' embedded-client devices to adhere to the feature's policies. When you publish a network feature, you have the option to push it right away or to push it later through provisioning or to individual devices through customer service. When you publish a network feature, it becomes available in the Provisioning list and in the list of Published network features in Customer Support records. 

Publishing an eCommerce policy means making it available to be added to products.

A network feature or eCommerce policy must be in Testing mode to be published. 

  1. Find the feature in Testing mode you want to publish.

    • For network features, go to Policy -> Feature.
    • For eCommerce policies, go to Policy -> eCommerce Policies.

  2. Click the corresponding View button  in the right column. (The buttons in the right column appear when you hover the mouse pointer over the row.)

  3. Click the Lock button  (to unlock and enable editing).

  4. Click Publish.
  5. If you are publishing a network feature, select the default policy condition:
    • Feature will be received by and enabled on devices when they next connect to the network or are reset. The default for the feature is On.
    • Feature will be received by a device only when it is enabled by Customer Support or during provisioning. The default for the feature is Off.
  6. Click Confirm.

Network Feature Default State

When you publish a network feature, you set its default state: Off or On. The Off state means that the feature is dormant on the ItsOn-powered device, and the On state means that the ItsOn-powered device adheres to the policy directives in the feature.  The Off/On default states are reflected in the Provisioning list, where the default state can also be changed. The default state can be overridden on one or more ItsOn-powered devices through the provisioning interface or on a single device through Customer Support.

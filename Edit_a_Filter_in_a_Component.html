Edit a Filter in a Component

Components are added to service policies in network features and to eCommerce policies, contain one or more filters and (sometimes) one or more policy events, and define the behavior of the network feature or eCommerce policy.

You can make changes to network features and eCommerce policies when they are in Draft mode (not published to the ItsOn server or to any embedded-client devices), in Test mode (when the network feature or eCommerce policy is pushed to embedded-client devices that you're using for testing) and in Published mode (when the network feature is already pushed to the ItsOn server and to the embedded-client devices of your customers or the eCommerce policy is made available to be added to products). Changes to network features are immediate, and are published to the ItsOn server immediately and to embedded-client devices as soon as they "check in" with the network (typically within 24 hours, or when the device is reset), or, when a network feature is moved from Draft to Test status, published to test devices immediately. Changes to eCommerce policies do not affect subscribers that already have subscriptions to products based on those eCommerce policies.

Exception: Once a bucket ID, rating group, or entitlement code has been added to a network feature or eCommerce policy, it cannot be deleted. It can only be exchanged, and only until the network feature or eCommerce policy has been published. Once a network feature or eCommerce policy has been published, the bucket ID, rating group, or entitlement code in the network feature or eCommerce policy can no longer be changed.

  1. Find the voice or service policy in Draft or Testing mode that you want to add a behavior policy to.
    • For a service policy in a network feature, Go to Policy > Features.
    • For a service eCommerce policy, go to Policy > eCommerce Policies.
  2. Click the corresponding View button  in the right column. (The buttons in the right column appear when you hover the mouse pointer over the row.)
  3. Click the Lock button  (to unlock and enable editing).
  4. Click the name of a filter in a service policy.
  5. Click the Lock button  in the Edit [Policy Type] Filter dialog box (to unlock and enable editing).
  6. Make any desired changes. You can:
    • Change the name of the filter (up to 32 characters).
    • Change the values of the existing data, voice, or messaging filter criteria.
    • Add a filter criterion (click Add Criteria, select the criterion you want to add, and then enter or select the value).
    • Delete a filter criterion (click the Delete icon next to the criterion you want to delete).
  7. Click Save.

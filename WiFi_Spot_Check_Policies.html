WiFi Spot Check Policies

This type of policy enables and controls spot-checking of the device's WiFi Internet connection. The goal of this type of policy is to fix problematic WiFi connections, and it applies only when WiFI is on and associated with a WiFI access point.

WiFI Spot Check policies have 3 settings and no policy events.

Setting Description
Check for Internet connectivity every ... when device is connected via WiFi Defines how often the device will check to see if it is connected to a WiFI access point and to the Internet. Select from 4, 2, or 1 (default) hours or 30, 20, 15, 10, or 5 minutes.
If Internet is unreachable, check again every ... up to ... times If the WiFi connection check fails, defines when the next check should be and how many times the device should check again before disabling the WiFI profile. Select from 10, 5, 4, 3, 2, or 1 (default) minutes or 30, 20, 15, 10, or 5 seconds and from 5, 4, 3 (default) 2, or 1 times.
If Internet is still unreachable, disable current WiFi profile for ... If a WiFi connection cannot be established, defines how long to disable the WiFi profile. After the timeout, the device will attempt to re-establish WiFi connectivity again according to policy. Select from 2 or 1 minutes or 30 (default), 20, 15, or 10 seconds.

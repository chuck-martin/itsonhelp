Edit a Data Session Policy in a Network Feature

You can make changes to network features and eCommerce policies when they are in Draft mode (not published to the ItsOn server or to any embedded-client devices), in Test mode (when the network feature or eCommerce policy is pushed to embedded-client devices that you're using for testing) and in Published mode (when the network feature is already pushed to the ItsOn server and to the embedded-client devices of your customers or the eCommerce policy is made available to be added to products). Changes to network features are immediate, and are published to the ItsOn server immediately and to embedded-client devices as soon as they "check in" with the network (typically within 24 hours, or when the device is reset), or, when a network feature is moved from Draft to Test status, published to test devices immediately. Changes to eCommerce policies do not affect subscribers that already have subscriptions to products based on those eCommerce policies.

Exception: Once a bucket ID, rating group, or entitlement code has been added to a network feature or eCommerce policy, it cannot be deleted. It can only be exchanged, and only until the network feature or eCommerce policy has been published. Once a network feature or eCommerce policy has been published, the bucket ID, rating group, or entitlement code in the network feature or eCommerce policy can no longer be changed.

You can:

  • Change the Data Session Trigger
  • Change the Criteria in the Default Policy Event
  • Change Blocked Network Groups

Change the Data Session Trigger

In Policy Settings, select an item from the Trigger drop-down list:

  • Screen Turn On (least cost savings): Policy is evaluated when the device screen is turned on.
  • Device Unlock: Policy is evaluated when the device is unlocked.
  • Internet App Brought to Foreground: Policy is evaluated when any app that can access the Internet is brought to the foreground.
  • None (most cost savings): Policy will always be evaluated.

Change the Criteria in the Default Policy Event

  • To remove a criteria, click its Remove  icon.
  • To add a criteria, click Criteria, then click the one you want to add. If you add multiple criteria, any one must be true when the policy is evaluated to establish a data session.

Change Blocked Network Groups

  • To add a network group:
    1. Click the Add button , then click New to create a new network group or From Library to use an already defined network group.
      • New: See Add a Network Group for details on creating new network groups.
      • From Library: In the Network Group list, select one or more network groups, and then click Add.
  • To remove a network group,  click its Remove  icon.

  • To change the details of a network group, click it's name and make the desired changes in the Edit dialog box. See Edit a Network Group for details.

    Changes you make to a network group will propagate to any other policies that use the network group.

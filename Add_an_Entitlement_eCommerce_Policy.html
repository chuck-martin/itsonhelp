Add an Entitlement eCommerce Policy

You can add an entitlement eCommerce policy to Service Design Center in 2 ways:

  • Add a new, completely blank entitlement eCommerce policy and configure it from scratch.
  • Add a new entitlement eCommerce policy as a copy of an existing entitlement eCommerce policy and modify the configuration of the copy.

The latter is especially helpful if you have existing entitlement eCommerce policies that are complex and/or well tested and you want to make only minor changes to a new one.

Add a New Blank Entitlement eCommerce Policy

  1. Go to Policy > eCommerce Policies.
  2. Click New >
    • Data > Entitlement Policy - to add a data entitlement eCommerce policy
    • Voice > Entitlement Policy - to add a voice entitlement eCommerce policy
    • Messaging > Entitlement Policy - to add a message entitlement eCommerce policy
  3. Enter a descriptive Name and optional Description. (Give it a meaningful name. This will help you find the policy when you are choosing from lists of policies.) The limit is 32 characters.
    Best Practice: Make the name of the policy the same as or similar to the entitlement code you add. Entitlement code names ideally reflect their functionality, so by giving the policy the same name, you'll be able to identify that functionality when you add the entitlement policy to an entitlement product.
  4. Click Save.
  5. Select a Priority.
  6. Next to Entitlement Code, click None > From Library.
  7. Find the entitlement code you want to add, select it, and then click Add. An entitlement code is a bucket ID, a single SOC, or multiple SOCs.
  8. Optionally add one or more components.
  9. Click the Unlock button  (to save and lock).

Add a Copy of an Existing Service eCommerce Policy

  1. Go to Policy > eCommerce Policies.
  2. Click New >
    • Data > Copy Entitlement Policy From - to select an existing data entitlement eCommerce policy
    • Voice > Copy Entitlement Policy From - to select an existing voice entitlement eCommerce policy
    • Messaging >  Copy Entitlement Policy From - to select an existing message entitlement eCommerce policy
  3. Find the policy you want to make a copy of, click to select it, and then click Add.
  4. Enter a new policy name to change it from the default name. The default name is something like [Policy Name] (Copy), where [Policy Name] is the name of the original entitlement eCommerce policy. (Give it a meaningful name. This will help you find the policy when you are choosing from lists of policies.) The limit is 32 characters.
  5. Click Save Copy.
  6. Make any desired changes.
  7. Click the Unlock button  (to save and lock).
Changes are saved automatically as you make them. However, an entitlement eCommerce policy is not available to be added to entitlement products until you test and publish the entitlement eCommerce policy.

Test a Network Feature or eCommerce Policy

All policies in a network feature or eCommerce policy must be valid and at least one embedded-client subscriber number must be entered in the Test Subscribers field before you can test a network feature or eCommerce policy. For policies to be valid, service policies must have at least one filter and at least one policy event, and data session policies must have at least one network group defined. If a network feature or eCommerce policy has only network enforcement, then the network feature or eCommerce policy cannot be tested (because testing requires embedded client subscribers), but the network feature or eCommerce policy can still be put into Test mode because being in Test mode is a prerequisite to publish a network feature or eCommerce policy.
  1. Edit a network feature or eCommerce policy in Draft mode. (Find the feature, click its View button , and then click the Unlock button .)
  2. In Test Subscribers, add subscriber lines of service by either entering them directly into the text box or by adding subscriber groups:
    • To add lines of service directly into the text box, click in a blank area and enter one or more phone numbers. (Enter 10 or 11 digits.)
    • To add subscriber groups
      1. Click Add From Group. 
      2. Select New to create a new set of subscribers or From Library to use an already defined group.
        • See Add a Subscriber Group for details on creating new subscriber groups.
        • In the list of all your Subscriber Groups, select one or more groups, and then click Add. The phone numbers from the groups are copied into the Test Groups field.
  3. Click Test (for testing) to push the feature to devices for testing. 
  4. Click Confirm. The policy or feature goes to Testing mode. 

The network feature or eCommerce policy is immediately pushed to the defined devices. 

When you are finished testing, either click Stop Testing to remove the feature from the defined devices or Publish when your testing confirms that the feature does what you want it to do and you want it pushed to the ItsOn network and to all your embedded-client devices.

Add an Entitlement Policy to a Network Feature

You can add an entitlement policy to a network feature in 2 ways:

  • Add a new, completely blank entitlement policy and configure it from scratch.
  • Add a new entitlement policy as a copy of an existing entitlement policy and modify the configuration of the copy.

The latter is especially helpful if you have existing entitlement policies that are complex and/or well tested and you want to make only minor changes to a new one.

Add a New Entitlement Policy to a Network Feature

  1. Click Add Policy >
    • Data > Entitlement Policy - to add a data entitlement policy
    • Voice > Entitlement Policy - to add a voice entitlement policy
    • Messaging > Entitlement Policy - to add a messaging entitlement policy
  2. Click on the policy name to change it from the default name. The default name is something like New [Policy Type] Entitlement Policy #, where [Policy Type] is either VOICE, DATA, or MESSAGING and # is a number. (Give it a meaningful name. This will help you find the policy when you are choosing from lists of policies.) The limit is 32 characters.
    Best Practice: Make the name of the policy the same as or similar to the entitlement code you add. Entitlement code names ideally reflect their functionality, so by giving the policy the same name, you'll be able to identify that functionality when you add the entitlement policy to an entitlement product.
  3. Next to Entitlement Code, click None > From Library.
  4. Find the entitlement code you want to add, select it, and then click Add. An entitlement code is a bucket ID, a single SOC, or multiple SOCs.
  5. Optionally add one or more components.
  6. Click the Unlock button  (to save and lock).

Add a Copy of an Existing Entitlement Policy to a Network Feature

Typically, you'll add a copy of an existing entitlement policy to take one that you've already created and modify/enhance it so you don't have to re-do work that has already been done. The value in this is if you have an existing entitlement policy that has one or more components and you want a new entitlement policy with similar component configuration, you can just modify the copy, rather than build a new one from scratch.

  1. Click Add Policy >
    • Data > Copy Entitlement Policy From - to add a copy of a data entitlement policy
    • Voice > Copy Entitlement Policy From - to add a copy of a voice entitlement policy
    • Messaging > Copy Entitlement Policy From - to add a copy of a messaging entitlement policy
  2. Find the policy you want to make a copy of, click to select it, and then click Add.
  3. Enter a new policy name to change it from the default name. The default name is something like New [Policy Type] Entitlement Policy # (Copy), where [Policy Type] is either VOICE, DATA, or MESSAGING and # is a number. (Give it a meaningful name. This will help you find the policy when you are choosing from lists of policies.) The limit is 32 characters.
    Best Practice: Make the name of the policy the same as or similar to the entitlement code in the entitlement policy. Entitlement code names ideally reflect their functionality, so by giving the policy the same name, you'll be able to identify that functionality when you add the entitlement policy to an entitlement product.
  4. Click Save Copy.
  5. Make any desired changes.
  6. Click the Unlock button  (to save and lock).

Device Capability and Functionality

The ItsOn integration into your cellular network system contains two primary components: the server-side software and the client-side software. The client-side software runs on devices that connect to your network. The client-side software is not required on devices that connect to your network for ItsOn services to function (unless you are a tenant with client enforcement), but the client-side software does provide additional capability and functionality. This topic explains that capability and functionality.

There are three client scenarios, typically referred to as embedded client, downloadable client, and no client.

Functionality Embedded Client Downloadable Client No Client
Description Embedded client means that the ItsOn software is embedded into the device ROM (read only memory) and integrated into the device operating system. Downloadable client means that the ItsOn software is not integrated into the device but run as an app. Devices may or may not be acquired by customers with the app already installed. If the device does not have the app pre-installed, it can be downloaded and installed separately. No client means that there is no ItsOn software on the device at all. The downloadable client may be able to be installed, but the embedded client cannot be.

ItsOn server contains configuration of published publishable features, and ability to connect to and operate on cellular network is controlled on the server level and is enforced when device attempts cellular network activity.

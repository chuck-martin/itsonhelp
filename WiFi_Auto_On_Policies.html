WiFi Auto On Policies

WiFi Auto On policies are designed to force a device to use WiFi networks instead of cellular networks whenever possible, even on the device where the user has specifically turned the device WiFi radio off.  WiFi Auto On policy event(s) are evaluated whenever there is a network transition event and whenever any of the selected criteria occur. If the policy event action is Enable and the device WiFi radio is turned off, the device WiFi radio is turned on.

A WiFi auto on policy has one policy event and one settings section.

WiFi Auto On Policy Event Criteria

  • Network Type: The policy event is evaluated whenever the device is in the defined type of network.
  • Time of Day: The policy event is evaluated only within the defined times and days.
  • Device is Stationary: The policy event is evaluated only when the device is stationary.

WiFi Auto On Settings

Setting Description
If this policy turned WiFi on, leave WiFi on even after policy no longer applies Enabled by default. If the device WiFi radio was turned on as a result of the policy event, enabling this setting causes the device WiFi radio to stay on even when the device is in a state where the policy event no longer evaluates to true.
Save Battery: If WiFi does not connect in...turn WiFi off for...then try again.

Enabled by default. Select 2 values:

  • How long the device should try to connect to WiFi if a connection cannot be made. Select from 1 (default), 2, 3, 4, 5 10 15, 30, or 50 minutes.
  • If a WiFi connection fails, how long to turn off the WiFi radio before turning it on to try connecting again. Select from 5 (default), 10, 15, or 30 seconds or 1 minute.

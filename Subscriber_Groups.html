Subscriber Groups

A subscriber is a line of service. Subscriber groups are simply groups of subscribers, defined by either phone numbers (also known as MDNs) or MSISDNs. In Service Design Center, they are used to make feature testing easier by allowing you to add multiple subscribers for testing with just a few clicks. 

The Subscriber Groups page lists the following details for each subscriber group:

  • Name
  • Number of subscribers
  • When the subscriber group was created
  • Who created the subscriber group

The default list order is by name alphabetically.

When you add a subscriber group as a test group, the subscribers from that group are added individually into the feature test area, rather than as a reference to the group. So if you make changes to a subscriber group, those changes won't propagate to the feature where you added the group to test. 

Valid Phone Numbers/MDNs

Service Design Center automatically validates when you save the subscriber group, and you are not allowed to save a subscriber group if it contains any invalid phone numbers/MDNs. See Validation and Fixing Data Errors for more information on entering valid data and fixing invalid data.

Icon

Validity is not the same as existence. Just because a phone number is valid does not mean it is one you manage. If you are using a group of devices for testing, make sure you enter the numbers correctly.
